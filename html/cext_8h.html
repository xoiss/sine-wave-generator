Detailed Description

Definitions of C language extensions.

This file provides definitions for different useful utilities, mostly macros, used as an ISO/ANSI C language extensions. These extensions are compatible (I hope) with language standards through C89, C90, C99, C11 including their GNU versions and compiler options -ansi, -pedantic, -Wall.

§ CONCAT

#define CONCAT (   pm,
  sm 
)    CONCAT_TOKENS(pm, sm)

Composes a single contiguous identifier from two tokens or macros associated to tokens.

Parameters
[in]pm– a token or a macro associated to a token that will be used as the prefix of the composed identifier.
[in]sm– a token or a macro associated to a token that will be used as the suffix of the composed identifier.
Returns
Identifier composed as the literal concatenation of tokens associated with pm and sm.

The pm must be a valid identifier or must be associated directly or indirectly with a valid identifier. The sm must itself be or must be associated either with a valid identifier, or a trailing part of a valid identifier – i.e., it may be associated with a numeric, or a single token composed from a numeric followed by a valid identifier, for example: _foo, 123, 777_bar.

Different macros may be used directly both for pm and sm, for example __COUNTER__ or __LINE__ may be used for sm.

Note
If a macro is specified either for pm or sm, it may be associated directly or indirectly with an allowed token, and the expansion will be followed through the whole chain of definitions until the final token. However, the final and all the intermediate associations must be defined in the translation unit above the place where the CONCAT macro is instantiated; otherwise the macro expansion process will stop in the middle and pm or sm respectively will be assigned with the intermediate macro's identifier instead of the final token.

Composes a single contiguous identifier from two tokens.

Parameters
[in]pt– a token that will be used as the prefix of the composed identifier.
[in]st– a token that will be used as the suffix of the composed identifier.
Returns
Identifier composed as the literal concatenation of two tokens: pt and st.

The pt must be a valid identifier. The st must be either a valid identifier, or a trailing part of a valid identifier – i.e., it may be a numeric, or a single token composed from a numeric followed by a valid identifier, for example: _foo, 123, 777_bar.

Note
If a macro is specified either for pt or st (which in turn returns an allowed token), it is not expanded – i.e., namely the macro's own identifier will be used but not the token string which that macro is associated to. Use CONCAT in this case instead of CONCAT_TOKENS.

§ static_assert

#define static_assert (   cond)    static_assert_msg(cond, _)

Validates the asserted expression during the compile time and puts an error.

Parameters
[in]cond– an asserted condition.

The asserted condition cond shall be computable at the compile time and evaluate to an integer constant. It shall evaluate to a non-zero value if the condition holds, and to zero otherwise.

If the cond does not hold, the compiler logs a compile time error.

This macro may be used both outside and inside a function definition block. The number of occurrences in the same translation unit and even in the same line of code is not limited.

Note
Under the ANSI C89 / ISO C90 standard, this macro shall be used before the first line of code in a code block. Note that even an empty instruction (i.e., an alone ';' token) or empty code block (i.e., a couple of braces) forms the line of code. If it is necessary to use this macro in the middle of the code block, it shall be surrounded with braces forming a nested code block.

§ static_assert_msg

#define static_assert_msg (   cond,
  msg 
)    struct CONCAT(__uid, __COUNTER__) { int msg : !!(cond); }

Validates the asserted condition during the compile time and puts the error message.

Parameters
[in]cond– an asserted condition.
[in]msg– an error message identifier.

The asserted condition cond shall be computable at the compile time and evaluate to an integer constant. It shall evaluate to a non-zero value if the condition holds, and to zero otherwise.

The msg must be specified with a valid identifier. Note, it must not be a sting literal.

If the cond does not hold, the compiler logs a compile time error with the message msg. The exact format of the error message depends on the compiler and the language standard. However, the msg will be included into it in quotes. If msg is not needed, the static_assert shall be used instead of static_assert_msg.

This macro may be used both outside and inside a function definition block. The number of occurrences in the same translation unit and even on the same line of code is not limited.

Note
Under the ANSI C89 / ISO C90 standard, this macro shall be used before the first line of code in a code block. Note that even an empty instruction (i.e., an alone ';' token) or empty code block (i.e., a couple of braces) forms the line of code. If it is necessary to use this macro in the middle of the code block, it shall be surrounded with braces forming a nested code block.
